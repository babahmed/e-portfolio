Information System Failure

A huge number of mobile telecommunications subscribers across the country who have been seeking to retrieve their lost Subscriber Identity Module (SIM) cards or acquire new lines were unable to do so due to the unavailability of the National Identity Management Commission’s NIN Verification Service (NVS) in the early week of February 2022.

The affected NIMC portal which is faced with technical glitches, enables telecom firms, the Nigerian Immigration Service, banks, and other organizations to verify the National Identity Number of their customers before attending to them, in line with the Federal Government’s directive.

Responding to complaints from users, MTN Nigeria said the “NIMC application is currently unavailable. Please bear with us as we are unable to process sim swap and upgrade requests due to external challenges. We appreciate your understanding and will post an update once this has been resolved.’’

The failure of this system almost grounded the economy to almost a halt impacting both corporate organizations and individuals who needed to swap their lost or stolen sim, Renew or obtain new international passports or verify onboarding for new users on various platforms.

Chike, O. (2022) NIMC portal temporarily unavailable as telecom firms, banks, others suffer service failure. nairametrics. Available from: https://nairametrics.com/2022/02/07/nimc-portal-temporarily-unavailable-as-telecom-firms-banks-others-suffer-service-failure/ [Accessed 20 Feb 2022].

Alternatives To SQL

The main difference between an SQL and a NoSQL database lies in how they are structured. A relational database—based in SQL—depends on the strictly defined relationships between tables. To retrieve information contained in the database, a user can refer to the specific search query language to match common characteristics across one or more tables. With a non-relational (NoSQL) database, there is no structural relationship or set of tables. Instead, the information is kept in separate JSON documents. While the data models vary from one NoSQL database to another, these databases are easy to use, scalable, resilient, and widely available. (TEC Team, 2018)

Relational databases were originally built on mainframes and were intended for accounting and other financial needs where data must be highly structured. MySQL is an example of how some SQL databases remain popular for applications with well-defined needs and uniform data types, thanks to their open-source nature. Relational database software is still widely available from several companies including Oracle, Microsoft, and Sybase. On the other hand, Non-relational and distributed data became popular as data needs became more complicated and expanded with the growth of the Internet. Instead of storing data in structured tables, a non-relational database stores it in separate documents which can take almost any format—usually JSON files. These databases are preferred in applications where structured data is not needed. There are a growing number of DBMS for non-relational databases, including MongoDB, HBase, and Cassandra DB. (TEC Team, 2018)

If the requirement has a fixed structure and doesn’t need frequent modifications, SQL is a preferable database. However, if you have applications where data is changing frequently and growing rapidly like in Big Data Analytics, NoSQL is arguably the best option.

TEC Team, 2018. A Dive Deep into Types of NoSQL Databases. [Online]

Available at: https://www3.technologyevaluation.com/research/article/sql-vs-nosql-databases-whats-the-difference.html [Accessed 24 01 2022].
